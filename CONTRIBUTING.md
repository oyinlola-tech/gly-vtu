# Contributing Guidelines

## Proprietary Access
This repository is proprietary and not free to use. Contributions are only accepted from authorized collaborators with explicit permission from the owner.

## How to Contribute (Authorized Only)
1. Discuss the change with the owner before starting.
2. Work on a feature branch.
3. Keep changes scoped and well-documented.
4. Update or add tests where applicable.
5. Update relevant documentation (`README.md`, `API.md`, `CHANGELOG.md`).

## Code Style
- Prefer clear, explicit code over cleverness.
- Avoid breaking changes without approval.
- Keep security controls intact (rate limits, CSRF, CORS, auth middleware).

## Reporting Issues
Use private channels only. Do not create public issues for security concerns.
